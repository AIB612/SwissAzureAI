# FINMA Compliance Checklist for AI in Financial Services
# Swiss Financial Market Supervisory Authority

## 1. Operational Risk Management (Circular 2023/1)

### 1.1 AI Model Governance
- [ ] AI model inventory maintained
- [ ] Model risk classification (high/medium/low)
- [ ] Model validation framework
- [ ] Independent model review
- [ ] Model performance monitoring
- [ ] Model change management process

### 1.2 Documentation Requirements
- [ ] Model development documentation
- [ ] Training data documentation
- [ ] Model assumptions documented
- [ ] Limitations clearly stated
- [ ] Version control implemented

### 1.3 Testing & Validation
- [ ] Back-testing performed
- [ ] Stress testing completed
- [ ] Sensitivity analysis
- [ ] Out-of-sample testing
- [ ] Ongoing monitoring metrics

## 2. Outsourcing Requirements (Circular 2018/3)

### 2.1 Cloud Provider Due Diligence
- [ ] Provider risk assessment
- [ ] Financial stability check
- [ ] Security certifications verified (SOC2, ISO27001)
- [ ] Data location confirmed (Switzerland)
- [ ] Subcontractor assessment
- [ ] Exit strategy defined

### 2.2 Contractual Requirements
- [ ] Service level agreements (SLA)
- [ ] Audit rights included
- [ ] Data protection clauses
- [ ] Termination provisions
- [ ] Business continuity requirements
- [ ] Incident notification obligations

### 2.3 Ongoing Monitoring
- [ ] Regular provider reviews
- [ ] Performance monitoring
- [ ] Compliance verification
- [ ] Security assessment updates

## 3. Business Continuity (Circular 2008/10)

### 3.1 AI System Availability
- [ ] Recovery time objective (RTO) defined
- [ ] Recovery point objective (RPO) defined
- [ ] Failover mechanisms tested
- [ ] Backup procedures documented
- [ ] Disaster recovery plan

### 3.2 Testing
- [ ] Annual DR testing
- [ ] Failover testing
- [ ] Data restoration testing
- [ ] Communication plan testing

## 4. Data Protection & Banking Secrecy

### 4.1 Banking Secrecy (Art. 47 Banking Act)
- [ ] Customer data classification
- [ ] Access controls for sensitive data
- [ ] Data never leaves Switzerland
- [ ] Encryption for all customer data
- [ ] Audit trail for data access

### 4.2 Data Handling
- [ ] Data minimization
- [ ] Purpose limitation
- [ ] Retention policies
- [ ] Secure deletion procedures

## 5. Cyber Security

### 5.1 Technical Controls
- [ ] Network segmentation
- [ ] Intrusion detection
- [ ] Vulnerability management
- [ ] Patch management
- [ ] Endpoint protection

### 5.2 Access Management
- [ ] Role-based access control
- [ ] Multi-factor authentication
- [ ] Privileged access management
- [ ] Regular access reviews

### 5.3 Monitoring & Response
- [ ] Security monitoring (SIEM)
- [ ] Incident response plan
- [ ] Forensic capabilities
- [ ] Regular penetration testing

## 6. Reporting Requirements

- [ ] Annual risk report
- [ ] Incident reporting procedures
- [ ] Material outsourcing notifications
- [ ] Model risk reporting
